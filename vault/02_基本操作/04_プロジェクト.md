v0.18.0（The Judgment Release、2026-07-01）でデスクトップアプリに加わった、コーディング作業のための「プロジェクト」機能の概要。

> [!note] 確認状況
> 本ページは v0.18.0 の[公式リリースノート](https://github.com/NousResearch/hermes-agent/releases/tag/v2026.7.1)と、実機（v0.18.0）と同一コミット `676236b` のデスクトップソース（サイドバーのプロジェクト実装・プロジェクトツリー・ターミナル・プレビューペーン）で確認した（2026-07-02）。プロジェクト操作の一部の画面文言は英語表示の場合がある。

## プロジェクトとは

これまでのデスクトップアプリはチャットが主役で、コードベースでの作業は [[03_設定/04_ワークスペース|ワークスペース]] の指定とチャット内のツール実行に頼っていた。v0.18 では、自分のコードベースをプロジェクトとして登録し、チャットのサイドバーから一覧・切り替えできる。

- **プロジェクトサイドバー**: チャットサイドバーにプロジェクトの管理が組み込まれ、プロファイルごとにコードベースを一覧・切り替えできる。**プロジェクトを作っていない既定状態では専用セクションは表示されない**（2026-07-02 実機画面で確認）。作成は Cmd+K パレットのプロジェクトコマンド（作成・フォルダ追加・プロジェクト概要の生成）から始める。
- **階層モデル**: `プロジェクト → リポジトリ → レーン` の階層。1 つのプロジェクトが複数リポジトリ・複数 worktree にまたがれる。
- **セッションのグループ化**: プロジェクトに属するセッションが、リポジトリ・worktree を横断してプロジェクト単位にまとまる。アーカイブから復元したセッションもプロジェクトのツリーへ戻る。
- **git worktree 管理**: プロジェクト内のリポジトリごとに worktree を検出・管理する（内部では `git worktree list` を並列に照会する）。

## 主な構成要素

| 要素 | 内容 |
|---|---|
| プロジェクト操作 | プロジェクトの作成、フォルダの追加、プロジェクト概要（IDEA.md）の生成。Cmd+K パレットからも操作できる |
| ターミナルパネル | 右サイドバーに複数ターミナルを開ける。タブとスクロールバックは再起動後も復元される。エージェントが使うターミナルは読み取り専用で観察できる |
| プレビューペーン | 右レールのプレビューでファイルの中身と変更（diff）を確認できる。チャット内には PR スタイルのファイル diff が表示される |
| リッチ埋め込み | アシスタントの Markdown に図・アラートなどのリッチ埋め込みをインライン表示する |

リモートゲートウェイのエージェントに対しても、フォルダピッカーと git 連携が機能する（ローカル専用ではない）。

## 関連する既存機能との関係

- **ワークスペース設定**: エージェントが読み書きする作業ディレクトリの考え方は [[03_設定/04_ワークスペース|ワークスペース]] を参照。プロジェクトはその上に載る「複数コードベースの管理と専用 UI」にあたる。
- **バックアップ**: 更新前スナップショットにプロジェクトのデータベース（`projects.db`）が含まれるようになった（v0.18）。
- **サブエージェントの観戦**: バックグラウンドで委譲した作業は、読み取り専用のスペクテータートランスクリプトでも追える。

## 失敗時の確認先

- プロジェクト機能が見当たらない場合は、まず [[03_設定/14_情報|情報]] でバージョンが v0.18.0 以上かを確認する。
- 更新しても表示が変わらない場合は、アプリを完全終了して開き直す（機能追加後の再起動が必要な場合がある）。
- 製品側の変更経緯は [[06_リファレンス/04_変更履歴|変更履歴]] を参照。
