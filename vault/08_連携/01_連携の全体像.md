> [!note] 確認状況
> v0.17 実機（ビルド `857d024`）のメッセージングパネルで確認。対応チャネルは多数あり、本ページは代表例と共通フローを示す。各チャネルの必須項目は接続先サービス側の仕様に依存する。

左ナビの「メッセージング」から、Hermes を各種チャットサービスに接続できる。接続すると、そのサービスの DM・グループ・チャンネルから Hermes を操作できるようになる。

## 対応チャネル（実機で確認）

Telegram / Discord / Slack / Mattermost / Matrix / WhatsApp / Signal / BlueBubbles（iMessage）/ Home Assistant / Email / SMS（Twilio）/ DingTalk / Feishu・Lark / WeCom（group bot・app）/ Weixin・WeChat / QQ Bot / Yuanbao（元宝）/ API server / Webhooks / Google Chat / IRC / Line / Msgraph Webhook / Ntfy / Photon / Raft / Relay（実験的）/ Simplex / Teams / Whatsapp Cloud。

- iMessage を使う場合は[[08_連携/02_iMessage|iMessage]]を参照（Photon / BlueBubbles の 2 経路）。
- Telegram・WhatsApp・Slack など主要チャネルの接続例は[[08_連携/03_メッセージング各種|メッセージング各種]]を参照。

## 共通の接続フロー

チャネルを選ぶと、右側に設定パネルが開く。多くのチャネルは次の流れで接続する。

1. **認証情報を取得**: パネルの案内に従い、サービス側でトークン等を取得する（「セットアップガイドを開く」リンクあり）。
2. **必須項目を入力**: ボットトークンなど、接続に必須の値を入力する。
3. **推奨項目を入力**: 許可ユーザー ID など、安全に使うための値を入力する。
4. **変更を保存**: 入力を保存する。
5. **トグルを ON**: チャネルを有効化する。

## 接続パターン（3種）

チャネルによって認証情報の渡し方が異なる。大きく次の3パターン。

1. **パネル入力型**: パネルに直接トークンや接続情報を入力する（Telegram・Discord・Slack・Signal・Email・Home Assistant・BlueBubbles など）。
2. **ローカルブリッジ型**: 同梱または別途のブリッジをローカルで起動して接続する（WhatsApp は QR スキャン、Signal は `signal-cli`、BlueBubbles は BlueBubbles サーバー）。
3. **セットアップガイド / CLI 型**: パネルの必須欄は「トークン不要」。外部（クラウドや CLI）で設定を済ませてから、パネルで許可ユーザーなどを入れてトグルを ON にする（LINE・Teams・Google Chat・Whatsapp Cloud・Photon・API server・Webhooks など）。

各チャネルの具体的な項目は[[08_連携/03_メッセージング各種|メッセージング各種]]を参照。

## 状態バッジとゲートウェイ

パネル上部に状態が表示される。

| バッジ | 意味 |
|---|---|
| 無効 | チャネルが OFF |
| 設定が必要 | 必須項目が未入力 |
| メッセージングゲートウェイが停止中 | チャネルを中継するゲートウェイが起動していない |

チャネルは「メッセージングゲートウェイ」を介して動く。ゲートウェイが停止中だと、設定済みでもメッセージは届かない（[[03_設定/10_ゲートウェイ|ゲートウェイ]]）。

## セキュリティ前提

> [!caution] 許可ユーザーの制限を必ず設定する
> 多くのチャネルでは、許可するユーザー ID を設定しないと「誰でも Bot に DM できる」状態になる。第三者があなたのエージェントを操作できてしまうため、推奨項目の許可ユーザー ID を必ず設定する。

- チャネル連携は外部からエージェントを操作する入口になる。承認モード（[[03_設定/05_安全性|安全性]]）と併せて、許可ユーザー・スコープを絞る。
- トークン等の認証情報は機密。共有・流出に注意する。

## 前提・操作・確認

- 前提: 接続先サービスのアカウントと、必要なトークン / API キーを用意できること。メッセージングゲートウェイが起動していること。
- 操作: チャネルを選び、認証情報を取得して必須・推奨項目を入力し、保存してトグルを ON にする。
- 期待結果: そのサービスから Hermes にメッセージを送ると応答が返る。
- 届かないとき: 状態バッジ（設定が必要 / ゲートウェイ停止中）、トークンの正否、許可ユーザー ID、ゲートウェイの起動を確認する。
