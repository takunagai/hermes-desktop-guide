> [!note] 確認状況
> v0.17 実機（ビルド `857d024`）で各チャネルの設定パネルを確認。日本で需要の高いチャネルを中心に、必須・推奨項目を抜粋する。既定値・項目名は実機表示に基づく。

共通の接続フロー・状態バッジ・セキュリティ前提は[[08_連携/01_連携の全体像|連携の全体像]]を参照。ここでは個別チャネルの接続項目をまとめる。iMessage（Photon / BlueBubbles）は[[08_連携/02_iMessage|iMessage]]を参照。

> [!note] 「必須」欄が「トークン不要」のチャネル
> LINE・Teams・Google Chat・WhatsApp・Whatsapp Cloud・API server・Webhooks などは、設定パネルの必須欄に「このプラットフォームはここでトークンが必要ありません。上のセットアップガイドを使用してから、以下で有効にしてください」と表示される。これらは「セットアップガイドを開く」から外部設定を済ませ、パネルでは許可ユーザーなどの推奨項目を入力してトグルを ON にする。

## LINE

LINE 公式アカウント（Messaging API）経由。認証情報は LINE Developers Console（Messaging API）で取得する。

| 項目 | 説明 |
|---|---|
| LINE channel access token | LINE Developers Console > Messaging API > Channel access token |
| LINE channel secret | Webhook 署名検証（HMAC-SHA256）に使う |
| Allowed user / group / room IDs | 許可する ID（カンマ区切り。`U`=ユーザー / `C`=グループ / `R`=ルーム 接頭辞） |
| Allow all users?（true/false） | 開発用。許可リストを無効化する |
| Home channel ID | cron / 通知の既定送信先 |
| Webhook host / port | 既定 `0.0.0.0` / `8646` |
| Public HTTPS base URL | 画像・音声・動画を LINE へ送る際に必要（例 `https://my-tunnel.example.com`）。bind アドレスが直接到達できない場合に必須 |
| Slow response threshold（秒） | 既定 45。`0` で無効 |

## Slack

Socket Mode 経由。Slack アプリを作成して Socket Mode を有効化し、workspace にインストールしてから 2 種のトークンを貼り付ける。

| 項目 | 必須 | 説明 |
|---|---|---|
| Slack ボットトークン | 必須 | OAuth & Permissions のボットトークン |
| Slack アプリトークン | 必須 | Socket Mode 用のアプリレベルトークン |
| 許可する Slack ユーザー ID | 推奨 | カンマ区切りの Slack ユーザー ID |

## Discord

DM・チャンネル・スレッドで利用。Discord Developer Portal でアプリ＋Bot を作成し、トークンを取得して適切なスコープでサーバーに招待する。

| 項目 | 必須 | 説明 |
|---|---|---|
| ボットトークン | 必須 | Discord Developer Portal のトークン |
| 許可する Discord ユーザー ID | 推奨 | カンマ区切り |
| 返信スタイル | 詳細 | `first` / `all` / `off` |
| すべての Discord ユーザーを許可 | 詳細 | 開発用。`true` で許可リストなしに誰でも DM 可 |
| ホームチャンネル ID / 名 | 詳細 | プロアクティブ送信先（Cron 出力・リマインダー）とその表示名 |

## Microsoft Teams

Azure AD アプリ（Bot Framework）として登録する。トークンはパネルでは不要で、セットアップガイドに従って Azure 側を設定する。

| 項目 | 説明 |
|---|---|
| Allowed users | 許可する Teams ユーザー ID / UPN（カンマ区切り） |
| Allow all users?（true/false） | 開発用 |
| Teams / Azure AD client ID | Bot Framework アプリのクライアント ID |
| Teams / Azure AD client secret | クライアントシークレット |
| Teams / Azure AD tenant ID | テナント ID |
| Home channel / display name | cron / 通知の既定 chat・channel ID とその表示名 |
| Webhook port | Bot Framework 既定 `3978` |

## Email

IMAP / SMTP のメールボックス経由。専用メールボックスの利用を推奨。Gmail / Workspace は app password を作り `imap.gmail.com` / `smtp.gmail.com` を使う。

| 項目 | 説明 |
|---|---|
| Email address | 送受信に使うアドレス |
| Email password | アカウントパスワードまたは app password |
| IMAP host | 例 `imap.gmail.com` |
| SMTP host | 例 `smtp.gmail.com` |

いずれも必須。

## Google Chat

GCP の Pub/Sub を介してイベントを受け取る。トークンはパネルでは不要で、セットアップガイドに従う。

| 項目 | 説明 |
|---|---|
| Allowed user emails | 許可するユーザーのメール（カンマ区切り） |
| Home space ID | cron / 通知の既定 space（例 `spaces/AAAA...`） |
| GCP project ID | Chat イベントの Pub/Sub をホストする GCP プロジェクト。未設定時は `GOOGLE_CLOUD_PROJECT` |
| Path to SA JSON | サービスアカウント JSON のパス。空なら Cloud Run / GCE の ADC。未設定時は `GOOGLE_APPLICATION_CREDENTIALS` |
| Pub/Sub subscription name | `projects/<proj>/subscriptions/<sub>` |

## Telegram

DM・グループ・トピックで利用。`@BotFather` でボットを作成してトークンを取得し、数値ユーザー ID は `@userinfobot` で得る。

| 項目 | 必須 | 説明 |
|---|---|---|
| ボットトークン | 必須 | `@BotFather` で `/newbot` |
| 許可する Telegram ユーザー ID | 推奨 | `@userinfobot` の数値 ID（カンマ区切り）。未設定だと誰でも DM 可 |
| プロキシ URL | 詳細 | Telegram がブロックされたネットワークでのみ必要 |

## WhatsApp

Hermes 同梱の WhatsApp ブリッジを起動し、初回に QR コードをスキャンして認証する（パネルでのトークン入力は不要）。

| 項目 | 説明 |
|---|---|
| 許可する WhatsApp ユーザー | 電話番号または WhatsApp ID（カンマ区切り） |
| WhatsApp ブリッジを有効にする | 下のトグルで自動設定される |
| ブリッジモード | WhatsApp ブリッジの動作モード |

公式の WhatsApp Cloud API を使う場合は別チャネル「Whatsapp Cloud」を選び、セットアップガイドに従う。

## Signal

`signal-cli` の REST ブリッジを別途起動し、その URL と登録電話番号を Hermes に指定する。

| 項目 | 必須 | 説明 |
|---|---|---|
| Signal ブリッジ URL | 必須 | 実行中の signal-cli REST ブリッジ（例 `http://127.0.0.1:8080`） |
| 電話番号 | 必須 | signal-cli ブリッジに登録した番号 |
| 許可する Signal ユーザー | 推奨 | Signal 識別子（カンマ区切り） |

## Home Assistant

Home Assistant 経由でスマートホームを操作したり、HA から Hermes に話しかけたりできる。HA のプロフィールで long-lived access token を作成して貼り付ける。

| 項目 | 説明 |
|---|---|
| Home Assistant URL | 既定 `http://homeassistant.local:8123` |
| Home Assistant Long-Lived Access Token | HA プロフィールで発行 |

いずれも必須。ツールセットの「Home Assistant」（デバイス操作）とは別に、チャネルとしても接続できる（[[07_ダッシュボード/02_スキルとツール|スキルとツール]]）。

## 開発者向け: API サーバー / Webhook

### API server（OpenAI 互換 API）

Hermes を OpenAI 互換の HTTP API として公開し、Open WebUI / LobeChat などのフロントエンドから接続する。

| 項目 | 説明 |
|---|---|
| Enable API server（true/false） | OpenAI 互換 API サーバーを有効化 |
| API server auth key | 認証用 Bearer トークン。有効時は必須で、未設定だと起動しない |
| API server port | 既定 `8642` |
| API server host | 既定 `127.0.0.1`。loopback でも auth key は必須 |
| API server model name | `/v1/models` で広告するモデル名。既定はプロファイル名 |

### Webhooks

GitHub・GitLab などから POST イベントを受信する HTTP サーバー。

| 項目 | 説明 |
|---|---|
| Enable webhooks（true/false） | Webhook アダプターを有効化 |
| Webhook port | 既定 `8644` |
| Webhook secret | 署名検証用のグローバル HMAC シークレット。`config.yaml` でルート別に上書き可 |

## その他のチャネル

メッセージングパネルには、上記以外にも次のチャネルがある（日本では需要が薄めのため、ここでは名前のみ。各パネルの「セットアップガイドを開く」に従う）。

- グループウェア / チャット: Mattermost、Matrix、DingTalk、Feishu・Lark、WeCom（group bot / app）、Weixin・WeChat、QQ Bot、Yuanbao（元宝）
- メッセージ / 通知 / 連携基盤: SMS（Twilio）、IRC、Msgraph Webhook、Ntfy、Relay

## 前提・操作・確認

- 前提: 接続先サービスのアカウントと、必要なトークン / API キー / サーバーを用意できること。メッセージングゲートウェイが起動していること。
- 操作: チャネルを選び、案内に従って認証情報を取得・入力し、保存してトグルを ON にする。許可ユーザーを必ず絞る。
- 期待結果: そのサービスから Hermes にメッセージを送ると応答が返る。
- 届かないとき: [[08_連携/01_連携の全体像|連携の全体像]]の「届かないとき」を参照。
