> [!note] 確認状況
> 通知パネルの存在・種別トグル・日本語 UI ラベルを v0.17（ビルド `857d024`）の実機とソース（`i18n/ja.ts` の `notifications` ブロック）で確認済み。

設定の「通知」パネルでは、OS のネイティブ通知を有効化し、どの種類のイベントで通知するかを種別ごとに切り替える。長時間の自動実行を任せているときに、承認待ちや完了を取りこぼさないための設定。

## マスタートグル

UI ラベルは「通知を有効にする」。通知全体の ON / OFF で、既定は ON。OFF にすると種別設定に関わらずすべての通知が出ない。

## 種別ごとのトグル（per-type）

イベント種別ごとに通知の有無を切り替える。いずれも既定 ON。

| UI ラベル | 内部キー | 通知されるタイミング |
|---|---|---|
| 承認が必要 | `approval` | ツール実行などの許可を求めるとき |
| 入力が必要 | `input` | エージェントがユーザーの入力を必要とするとき |
| 応答完了 | `turnDone` | 応答（ターン）が完了したとき |
| ターン失敗 | `turnError` | 実行が失敗したとき |
| バックグラウンドタスク完了 | `backgroundDone` | バックグラウンドのサブエージェント等が終わったとき |

## 完了サウンドとテスト通知

- **完了サウンド**: エージェントのターン終了時に鳴らすサウンドをプリセットから選び、「試聴」で確認できる。
- **テスト通知**: 「テスト通知を送信」で、設定した表示とサウンドを実際に確認できる。

## 前提・操作・確認

- 前提: OS 側で Hermes Desktop の通知が許可されていること（macOS は「システム設定 > 通知」、Windows は「設定 > システム > 通知」）。
- 操作: マスタートグルを ON にし、必要な種別を選び、サウンドを選んで保存する。テスト通知で表示を確認する。
- 期待結果: 選んだ種別のイベントで OS 通知が表示される。
- 通知が出ないとき: OS の通知許可、マスタートグル、対象種別のトグル、OS のおやすみモード / 集中モードを確認する。
