> 通知パネルの存在と種別トグルは v0.17 実機（ビルド `857d024`）で確認済み。各トグルの日本語 UI ラベルの正確な文言は要再確認のため `needs-review`。

設定の「通知」パネルでは、OS のネイティブ通知を有効化し、どの種類のイベントで通知するかを種別ごとに切り替える。長時間の自動実行を任せているときに、承認待ちや完了を取りこぼさないための設定。

## マスタートグル

通知全体の ON / OFF。既定は ON。これを OFF にすると、種別設定に関わらず通知は出ない。

## 種別ごとのトグル（per-type）

イベント種別ごとに通知の有無を切り替える。いずれも既定 ON。

| 種別（内部キー） | 通知されるタイミング |
|---|---|
| `approval` | 承認待ち（ツール実行などの許可を求めるとき） |
| `input` | 入力待ち（エージェントがユーザーの入力を必要とするとき） |
| `turnDone` | ターン完了（応答が完了したとき） |
| `turnError` | ターンのエラー（実行が失敗したとき） |
| `backgroundDone` | バックグラウンド処理の完了（バックグラウンドのサブエージェント等が終わったとき） |

## 完了サウンドとテスト通知

- **完了サウンド**: 通知時に鳴らすサウンドを選択できる。
- **テスト通知**: 設定した表示とサウンドを実際に確認するためのテスト通知を送れる。

## 前提・操作・確認

- 前提: OS 側で Hermes Desktop の通知が許可されていること（macOS は「システム設定 > 通知」、Windows は「設定 > システム > 通知」）。
- 操作: マスタートグルを ON にし、必要な種別を選び、サウンドを選んで保存する。テスト通知で表示を確認する。
- 期待結果: 選んだ種別のイベントで OS 通知が表示される。
- 通知が出ないとき: OS の通知許可、マスタートグル、対象種別のトグル、OS のおやすみモード / 集中モードを確認する。
