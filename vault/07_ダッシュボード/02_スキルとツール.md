> [!note] 確認状況
> v0.17 実機（ビルド `857d024`）の「スキルとツール」パネルで、ツールセット 26 件（うち 19 件が有効）とスキル 103 件（19 カテゴリ）を確認した。スキルの件数・カテゴリ・有効数、各ツールセットの「設定済み」/「キーが必要」表示は、導入したスキルや API キーの設定状況によって変わる。ここに挙げる一覧は確認時点のスナップショット。

左ナビの「スキルとツール」は、「新しいセッション」「メッセージング」「アーティファクト」と並ぶトップレベル画面。エージェントが使える**スキル**と**ツールセット**を一覧し、トグルで有効 / 無効を切り替える。上部の「スキル」「ツールセット」タブで表示を切り替え、右上の検索ボックスで名前・説明から絞り込める。

> [!tip] スキルとツールセットの違い
> - **ツールセット**は、エージェントが直接呼び出す組み込みツール（関数）をまとめた単位。`web_search` や `read_file` のような具体的な操作で、含まれるツール名（タグ）が各行に表示される。
> - **スキル**は、特定の作業手順やノウハウを記述した拡張（SKILL.md）。有効にすると、その手順をエージェントが必要に応じて参照・実行できる。`apple-notes` や `github-pr-workflow` のように、外部 CLI やサービスの使い方をまとめたものが多い。
>
> どちらもトグルが ON のものだけがエージェントから使える。

## ツールセット

> [!note] 「設定済み」と「キーが必要」
> 各行の状態バッジは、そのツールセットを有効化できる状態かどうかを示す。**設定済み**は追加設定なしで（または必要なキーが既に揃っていて）ON にできる状態。**キーが必要**は、対応する API キー等を設定するまで有効化できない状態で、実機例では Home Assistant と Mixture of Agents が該当した。X (Twitter) Search のようにキーが要るツールセットでも、キー設定済みなら「設定済み」と表示される。つまりバッジは環境のキー設定状況で変わり、トグルの ON / OFF（実機例では 7 件が OFF）とは別物。

主なカテゴリの傾向は次のとおり。

- **Apple / Smart-Home / Media**: macOS のメモ・リマインダー・iMessage・FindMy、Philips Hue、Spotify や YouTube など、手元の端末・サービスを操作するスキル。
- **Autonomous-AI-Agents**: `claude-code`・`codex`・`opencode` など、別のコーディングエージェントへ実装を委譲するスキル群。
- **General / Github / Software-Development**: Cloudflare 開発（Workers・D1・R2・Durable Objects）、GitHub の PR / Issue 運用、計画立て（`plan`・`writing-plans`）・TDD・体系的デバッグなど、開発作業の手順。
- **Creative**: 図版・ASCII アート・インフォグラフィック・作曲・p5.js など、制作物を作るスキル。
- **Mlops**: vLLM や llama.cpp によるモデル提供、評価、HuggingFace 連携などの ML 運用。

> [!tip] スキルとは
> スキルは特定の作業手順やツール群をまとめた拡張。有効にしたスキルだけがエージェントから参照・実行できる。使う予定のないスキルは OFF にしておくと、意図しない動作や候補の混在を避けられる。

> [!caution] 安全性に関わるスキル
> 既定の一覧には、Red-Teaming カテゴリの `godmode`（LLM のジェイルブレイク手法）や Mlops の `obliteratus`（モデルの拒否応答を除去する手法）など、LLM の安全機構を回避する目的のスキルも含まれる。有効化しただけで自動実行されるわけではないが、意図せず使われないよう、必要なスキルだけを ON にし、用途が不明なものは OFF のままにしておく。

## 前提・操作・確認

- 前提: スキル / ツールセットの一覧が表示できること。
- 操作: 使いたいスキル・ツールセットのトグルを ON にする。「キーが必要」のものは先にキーを設定する。
- 期待結果: 有効にしたスキル・ツールがエージェントから使える。
- 使えないとき: トグルが ON か、「キーが必要」表示が残っていないか、対象の API キー設定を確認する（[[03_設定/11_ツールとキー|ツールとキー]]・[[03_設定/12_MCP|MCP]]）。
