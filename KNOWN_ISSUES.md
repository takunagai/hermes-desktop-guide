# 既知の問題

> **対象**: Hermes Desktop ガイド（https://hermes.ai-deck.app）
> **最終更新**: 2026-06-20

開発・保守上の既知の問題と、その判断・対処方針を記録する。サイト表示やビルドには影響しないが、Vault（ソース）を編集する人が知っておくべき事項を中心に扱う。

## 1. 設定ページの「表示順（サイドバー・索引番号）」と「ファイル接頭辞」のズレ

- **発生日**: 2026-06-20
- **状態**: 意図的に許容（現状維持）

### 内容

実機（v0.17 / コミット `857d024`）の設定画面の並びに合わせ、「通知」を「詳細」の次（9 番目）へ移動した。表示順は各設定ページ frontmatter の `sidebar.order` と「全設定項目索引」の見出し番号で制御し、**Vault のファイル名（接頭辞）は変更していない**。このため次のズレが生じている。

| 表示（サイドバー / 索引） | ファイル |
|---|---|
| 09 通知 | `vault/03_設定/15_通知.md` |
| 10 プロバイダー | `vault/03_設定/09_プロバイダー.md` |
| 11 ゲートウェイ | `vault/03_設定/10_ゲートウェイ.md` |
| 12 ツールとキー | `vault/03_設定/11_ツールとキー.md` |
| 13 MCP | `vault/03_設定/12_MCP.md` |
| 14 アーカイブ済みチャット | `vault/03_設定/13_アーカイブ済みチャット.md` |
| 15 情報 | `vault/03_設定/14_情報.md` |

### 影響

- **サイト表示・公開 URL には影響なし**。公開 URL は frontmatter `slug` 由来（例: `15_通知.md` → `/settings/notifications/`）で、ファイル接頭辞や表示番号とは独立している。
- 影響するのは Vault（Obsidian）でソースを編集する人のみ。Obsidian のファイル一覧の並びが表示順と一致せず、索引で `## 14 …` 見出しの直下に `[[03_設定/13_…]]` が並ぶため、一見不整合に見える。

### なぜ直さないか（現状維持の判断）

ファイル名を表示順に合わせてリネームするには、Vault 全体の該当 WikiLink（changelog・各ガイド・ダッシュボード・連携・索引など）を一括更新する必要があり、churn と破損リスクが大きい。サイトは正しく動作しているため、コスメティックな差のために広範な変更は行わない。

### 解消したい場合の手順

1. `vault/03_設定/` の 7 ファイルを表示順にリネームする（`09_プロバイダー` → `10_プロバイダー` … `15_通知` → `09_通知`）。
2. Vault 全体で該当ファイルへの WikiLink を新パスへ更新する。
3. `npm run preprocess` を実行する（未解決 WikiLink があれば停止するため検出できる）。アンカー参照（`…索引#NN 画面名`）も見出し番号に合わせて更新する。
4. `sidebar.order`・索引見出し番号・ファイル接頭辞が一致していることを確認する。
