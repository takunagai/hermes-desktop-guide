## 作業手順

### 1. 品質ゲート全通し（`content-validator` に委譲）
- `npm run quality`（= `lint && check && test && build && check:links`）。
- 落ちたら preprocess→test→check→build→check:links のどこで落ちたか切り分けて報告。軽微（誤字・リンクパス・アンカー）はその場で修正、内容起因は該当 Wave に差し戻し。
- 到達状態: `npm run quality` が全項目 pass。

### 2. ブラウザ確認（人間協働）
`npm run dev`（または `npm run preview`）で起動し、人間が確認:
- PageTitle のメタ表示: `v0.17.0`・`verified` 日付・status バッジ（緑=確認済み / 橙=要再確認）が正しく出る。
- 新規ページ/新セクションが **sidebar** に表示される（Wave 4 で追加した場合）。
- **Pagefind 全文検索**に新規/変更ページがヒットする。
- 前後ナビ・モバイル表示・キーボード操作（Tab/Skip Link）。
- 到達状態: 上記が問題なし（chrome-devtools MCP で自動確認してもよいが、最終目視は人間）。

### 3. status 棚卸し
- `verified` / `needs-review` の最終分布を確認。**needs-review が残るのは許容**（橙バッジで「要再確認」と明示されるため安全に公開できる）。残った needs-review ページとその理由を実行ログにまとめる。

### 4. 最終コミット（`git-version-control`）
- 未コミットの最終差分（Wave 5 の横串スイープ等）を Conventional Commits（日本語）でコミット。
- **AI 署名を付けない**（`Co-Authored-By` 等を入れない）。
- `.firecrawl/` `_handover/` を**含めない**（gitignore 済みのはず。`git status` で確認）。
- 到達状態: `git status --short` がクリーン（無視対象を除く）。

### 5. デプロイ判断（【人間】承認必須）
- `main` へマージ/push すると Cloudflare Workers Builds が本番（`https://hermes.ai-deck.app`）へ自動デプロイする。
- **人間の承認を得てから** マージ/push する。承認が出るまで `docs/hermes-v0.17` で停止。
- 非本番ブランチのプレビュー確認をしたい場合は `npx wrangler versions upload`（任意）。

## 失敗時の撤退

- `quality` が内容起因で落ちる → 該当 Wave のコミットを特定し、その Wave を再実行（`git revert` ではなく該当ファイルを直すか、該当 Wave 指示書を再起動）。各 Wave がコミット単位なので Wave 単位で巻き戻せる。
- デプロイ後に問題発覚 → Cloudflare Workers のロールバック（前 Version へ）。`main` の該当コミットを `git revert` して再 push。
