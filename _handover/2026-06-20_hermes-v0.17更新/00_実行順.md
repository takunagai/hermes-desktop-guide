### このガイドの構造（前提知識）

- コンテンツの正本は **`vault/`**。`src/content/docs/` は `scripts/preprocess.ts` が再生成する gitignore 対象の中間生成物。**直接編集しない**。
- 公開 URL は frontmatter `slug`（ASCII）から生成。**slug は変更しない**（変更すると `astro.config.mjs` の redirects 追加が必要になる）。
- frontmatter `status`（`verified`=緑バッジ / `needs-review`=橙「要再確認」バッジ）・`verified` 日付・`hermes_version`・`hermes_commit`・`tags` は `src/components/PageTitle.astro` が画面表示する。**frontmatter を直せばバッジは自動更新**されるが、本文プローズ中の版表記（`0.16.0` 等）は手編集が必要。
- サイドバーは `astro.config.mjs` の `autogenerate`（slug 先頭ディレクトリで走査）。**既存セクション内のページ追加は config 変更不要**（`sidebar.order` で制御）。**新セクション追加時のみ** `astro.config.mjs` の `sidebar` 配列に 1 エントリ追加が必須。
- 品質ゲート: `npm run quality` = `lint && check && test && build && check:links`。`npm run preprocess` は ContentValidationError（未解決 WikiLink・slug/basename 重複・frontmatter 欠落）で停止する。
- vault フォルダ: `01_はじめに/` `02_基本操作/` `03_設定/`（14画面）`04_ガイド/` `05_トラブルシューティング/` `06_リファレンス/`。`hermes_version: 0.16.0` を持つのは 20 ファイル（設定14 + ガイド3 + リファレンス3）。`はじめに`/`基本操作`/`トラブル`/`ホーム` の 6 ファイルは `hermes_version` 無し（`verified` のみ）。

### ゴールと安全側運用（最重要方針）

- ユーザー手元の v0.17 実機で全設定画面を再監査し、本文・frontmatter・全設定索引・調査根拠・新機能をガイドへ反映する。
- **実機確認できたページのみ `verified` を当日へ更新する。未確認は `status: needs-review` に落とす（橙バッジで「要再確認」と正直に表示）。推測で verified にしない。**
- 途中公開でも needs-review バッジにより誤情報の断定公開は起きない（安全側設計）。

### 横断する最重要ガード（過剰改変の防止）

v0.17 の破壊的変更とされる **`write_mode` / `write_approval` / `send_message` は、vault 全体に名前として 1 件も存在しない**（grep 0 件）。本ガイドは「Desktop の設定画面」を記述しており、CLI/エージェントレイヤの生 config・ツール名は載せていないため。
→ これらの破壊的変更は「既存記述の書き換え」ではなく、**「v0.17 の Desktop UI に新たに露出したかをソース＋実機で確認し、露出した時だけ加筆。露出しないなら変更履歴ページに移行注記」** として扱う。**既存の承認モード `approvals.mode`（manual/smart/off）等を、リリースノートの別レイヤ用語に引きずられて書き換えてはならない。** 迷ったら Stop And Ask。

## コミット方針（共通）

- **各 Wave は完了時に、その Wave の成果を `docs/hermes-v0.17` へコミットする**（`git-version-control` 委譲）。Wave 単位のコミットにより、別セッションの次 Wave がクリーンな状態から始められ、撤退も Wave 単位でできる。
